### 6. Despliegue en producción — `https://tmpmailio.com`

**Infraestructura configurada:**
- Instalación y configuración de **Nginx** como reverse proxy
- Proxy de rutas: `/api/*` y `/socket.io/*` → backend (`:4000`), resto → frontend (`:3000`)
- Soporte WebSocket en Nginx (headers `Upgrade` / `Connection`)
- Apertura de puertos 80 y 443 en UFW
- Certificado SSL gratuito **Let's Encrypt** via Certbot (válido hasta junio 2026)
- Renovación automática programada por Certbot
- Redirección automática HTTP → HTTPS

**Variables de entorno actualizadas:**
- `CORS_ORIGIN=https://tmpmailio.com`
- `NODE_ENV=production`
- `NEXT_PUBLIC_API_URL=https://tmpmailio.com`
- `NEXT_PUBLIC_SOCKET_URL=https://tmpmailio.com`
- `NEXT_PUBLIC_SITE_URL=https://tmpmailio.com`

**Resolución de Mixed Content:**
- Rebuild forzado del frontend (sin caché de Turbo) para hornear las nuevas URLs en el bundle

### Extensión para navegadores (Chrome / Firefox / Edge)

Desarrollar una extensión oficial de TmpMail que permita generar y gestionar buzones temporales directamente desde el navegador, sin necesidad de abrir la web.

**Funcionalidades propuestas:**

- [ ] **Popup principal** — interfaz compacta (300×400px) con la dirección activa, botón copiar y contador de tiempo restante
- [ ] **Generación de buzón** — crear un nuevo correo temporal con un clic desde cualquier pestaña
- [ ] **Notificaciones nativas** — alerta del sistema operativo cuando llega un correo nuevo (usando la API de Notifications del navegador)
- [ ] **Badge en el icono** — mostrar el número de correos no leídos sobre el icono de la extensión
- [ ] **Vista de bandeja** — listado de correos recibidos dentro del popup, con vista previa del asunto y remitente
- [ ] **Lectura de correos** — abrir el cuerpo del correo en un panel expandido o en una ventana emergente
- [ ] **Auto-fill** — detectar campos de tipo `email` en formularios web y ofrecer insertar el correo temporal con un clic
- [ ] **Historial de sesión** — recordar el buzón activo entre pestañas y reinicios del navegador (usando `chrome.storage.local`)
- [ ] **Selector de dominio** — elegir el dominio del correo generado desde el popup
- [ ] **Conexión en tiempo real** — mantener Socket.io activo en background para recibir correos sin tener la web abierta

**Stack técnico sugerido:**
- Manifest V3 (requerido por Chrome Web Store desde 2024)
- React + Vite para el popup UI
- `chrome.storage.local` para persistencia del buzón
- `chrome.notifications` para alertas nativas
- Service Worker como background script para Socket.io y polling
- API compartida con el backend actual (`https://tmpmailio.com/api`)
- Compatible con Chrome, Edge (Chromium) y Firefox (WebExtensions API)
